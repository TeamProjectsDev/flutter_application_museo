02 Arquitectura del Sistema

La aplicación sigue un modelo de arquitectura en capas con separación clara entre Presentation, Domain y Data.

Presentation Layer

UI declarativa con Flutter. Widgets reactivos desacoplados de la lógica.

Domain Layer

Lógica de negocio pura mediante Providers de Riverpod.

Data Layer

Servicios de Firebase, Supabase Storage y Stripe.

04 Gestión de Estado: Riverpod

Lógica Reactiva

// Provider que calcula el rango según las piezas desbloqueadas
final userRankProvider = Provider((ref) {
  final unlockedCount = ref.watch(unlockedPiecesProvider).length;
  if (unlockedCount >= 6) return UserRank.conservadorJefe;
  if (unlockedCount >= 3) return UserRank.academico;
  if (unlockedCount >= 1) return UserRank.explorador;
  return UserRank.visitante;
});
