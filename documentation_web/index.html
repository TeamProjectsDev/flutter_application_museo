01 Visión General del Proyecto

patrimonio del Museo Histórico Padre Suárez, integrando modelos 3D, gamificación, pagos con Stripe y sincronización en tiempo real con Firebase + Supabase.

Pilares Técnicos

  • ✓ Frontend: Flutter con Riverpod para state management
  • ✓ BD Tiempo Real: Firestore + Supabase (PostgreSQL)
  • ✓ Pagos: Stripe (Web, Móvil y Desktop)
  • ✓ Autenticación: Firebase Auth + OAuth Google
  • ✓ Almacenamiento: Cloud Storage + Cloudflare R2
  • ✓ Análisis: Firebase Analytics + Crashlytics

Características Clave

  • ✓ Escaneo QR integrado con modelos GLB 3D
  • ✓ Realidad Aumentada nativa (ARCore/ARKit)
  • ✓ Sistema de rangos gamificados (Visitante → Conservador)
  • ✓ Entradas digitales con códigos QR
  • ✓ Modo offline con sincronización
  • ✓ Multidioma (i18n) + Accesibilidad (a11y)

✨ Funcionalidades Comerciales

  • ✓ Mapa 3D Interactivo: Plano de planta de alta fidelidad con modelado 3D y adaptación dinámica de temas (Claro/Oscuro)
  • ✓ Bio-Revista RSS: Noticias científicas en tiempo real con caché local
  • ✓ Deep Links: Acceso directo a piezas mediante museo://pieza/[id]
  • ✓ Onboarding: Tutorial de bienvenida con persistencia de estado
  • ✓ Offline Mode: Catálogo cacheado para visitas sin red

02 Arquitectura del Sistema

La aplicación sigue un modelo de arquitectura en capas con separación clara entre Presentation (UI), Domain (Lógica) y Data (Servicios).

Explicación de Capas

📱 Presentation Layer: Widgets de Flutter y screens. Gestiona UI, navegación y eventos del usuario. Sin lógica de negocio.
⚙️ Domain Layer: Providers Riverpod, notifiers y use cases. Aquí vive toda la lógica. Independiente de la capa de datos.
💾 Data Layer: Servicios de Firebase (BD/Auth), Supabase Storage (Multimedia), Stripe y APIs externas.

03 Arquitectura de Datos

La aplicación utiliza una estrategia híbrida: Firebase Firestore para la lógica de negocio y estado del usuario en tiempo real, y Supabase Storage como servidor de activos multimedia (CDN) para modelos 3D e imágenes.

04 Gestión de Estado con Riverpod

Riverpod es el framework elegido para manejar el estado de la aplicación de forma reactiva, eficiente y testeable. Aquí detallamos la arquitectura.

Providers Principales

Provider Tipo Responsabilidad
authProvider StateNotifier Gestión de sesión (login, logout, Google Auth)
userProfileProvider FutureProvider Datos del usuario (Firebase Firestore)
piecesProvider FutureProvider Catálogo de piezas 3D (Supabase)
userUnlocksProvider StateNotifierProvider Piezas desbloqueadas (scan, donación)
paymentProvider StateNotifierProvider Orquestación de pagos Stripe
gamificationProvider StateNotifier Cálculo de rango y progreso
offlineCacheProvider StateNotifier Sincronización offline/online

Ejemplo: authProvider

final authProvider = StateNotifierProvider<
  AuthNotifier, 
  AsyncValue<User?>
>((ref) {
  return AuthNotifier(
    firebaseAuth: FirebaseAuth.instance,
    ref: ref,
  );
});

class AuthNotifier extends StateNotifier<AsyncValue<User?>> {
  AuthNotifier({
    required this.firebaseAuth,
    required this.ref,
  }) : super(const AsyncValue.data(null));

  final FirebaseAuth firebaseAuth;
  final Ref ref;

  Future<void> signInWithGoogle() async {
    state = const AsyncValue.loading();
    try {
      final GoogleSignInAccount? googleUser = 
        await GoogleSignIn().signIn();
      final GoogleSignInAuthentication? googleAuth = 
        await googleUser?.authentication;
      final credential = 
        GoogleAuthProvider.credential(
          idToken: googleAuth?.idToken,
          accessToken: googleAuth?.accessToken,
        );
      await firebaseAuth.signInWithCredential(credential);
      state = AsyncValue.data(firebaseAuth.currentUser);
    } catch (e, st) {
      state = AsyncValue.error(e, st);
    }
  }
}

Ejemplo: userUnlocksProvider

final userUnlocksProvider = 
  StateNotifierProvider<
    UserUnlocksNotifier, 
    AsyncValue<Set<String>>
  >((ref) {
  return UserUnlocksNotifier(
    ref: ref,
    supabaseClient: Supabase.instance.client,
  );
});

class UserUnlocksNotifier 
  extends StateNotifier<AsyncValue<Set<String>>> {
  
  Future<void> unlockPiece(String pieceId) async {
    final userId = ref.watch(authProvider)
      .value?.uid;
    
    if (userId == null) return;
    
    state = const AsyncValue.loading();
    try {
      await supabaseClient
        .from('user_unlocks')
        .insert({
          'user_id': userId,
          'piece_id': pieceId,
        });
      
      // Refrescar
      _loadUnlocks();
    } catch (e, st) {
      state = AsyncValue.error(e, st);
    }
  }
}

Flujo Reactivo: Ejemplo Práctico

Caso: Usuario escanea un QR, descubrimos pieza, actualizamos rango automáticamente.

  1. ar_screen.dart detecta QR y llama ref.read(userUnlocksProvider.notifier).unlockPiece(pieceId)
  2. UserUnlocksNotifier inserta en Supabase y actualiza estado local
  3. gamificationProvider observa cambios en userUnlocksProvider y recalcula rango
  4. profile_screen.dart escucha gamificationProvider y redibuja el rango automáticamente
  5. Todo sin navegación, sin setState, 100% reactivo

07 Flujos de Pago: Stripe Checkout

Estrategia unificada mediante Stripe Checkout para máxima seguridad y compatibilidad.

Plataforma Web (Stripe)

  1. Usuario donante pulsa "Donar"
  2. paymentProvider crea Checkout Session
  3. Stripe devuelve URL temporal
  4. App abre URL en navegador (webview)
  5. Usuario introduce tarjeta en Stripe Checkout
  6. Si exitoso, Stripe redirige a success_url
  7. App recibe confirmación y actualiza rango

Pasarela Única (Stripe)

  1. Usuario abre app en iOS/Android
  2. Pulsa "Donar" → abre sheet bottom
  3. Selecciona monto (3€, 5€, 10€)
  4. Stripe maneja el pago nativo y web
  5. Se abre cuadro Face ID / Touch ID
  6. Confirmación automática
  7. Webhook → Supabase → rango actualizado

Detección Automática de Plataforma

Future<void> processDonation(double amount) async {
  final isWeb = kIsWeb;
  final isIOS = defaultTargetPlatform == TargetPlatform.iOS;
  final isAndroid = defaultTargetPlatform == TargetPlatform.android;
  
  if (isWeb || !isIOS && !isAndroid) {
    // Usar Stripe en cualquier plataforma
    await ref.read(paymentProvider.notifier)
        .initiateStripeCheckout(amount);
  }
}

Ejemplo: Google Sign-In Implementation

Future<void> signInWithGoogle() async {
  try {
    // 1. Obtener Google account
    final GoogleSignInAccount? googleUser = await GoogleSignIn().signIn();
    if (googleUser == null) return; // Usuario canceló
    
    // 2. Obtener tokens
    final GoogleSignInAuthentication googleAuth = 
      await googleUser.authentication;
    
    // 3. Crear credential para Firebase
    final credential = GoogleAuthProvider.credential(
      idToken: googleAuth.idToken,
      accessToken: googleAuth.accessToken,
    );
    
    // 4. Sign in con Firebase
    final userCredential = await FirebaseAuth.instance
        .signInWithCredential(credential);
    
    final userId = userCredential.user?.uid;
    
    // 5. Crear/actualizar documento en Firestore
    if (userId != null) {
      await FirebaseFirestore.instance
          .collection('users')
          .doc(userId)
          .set({
            'email': googleUser.email,
            'displayName': googleUser.displayName,
            'photoUrl': googleUser.photoUrl,
            'createdAt': FieldValue.serverTimestamp(),
          }, SetOptions(merge: true));
    }
    
    // 6. Estado Riverpod se actualiza automáticamente
    // Los listeners reaccionan y actualizar UI
  } catch (e) {
    print('Google Sign-in error: $e');
  }
}
